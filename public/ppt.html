The Problem

企业后台开发，到底在浪费什么？

80% 的后台系统都是相同的 CRUD——但每次都要从零开始写 Controller、Service、DAO、前端表格、表单、权限……

重复劳动

每个管理模块：Controller × 5、Service × 3、DAO × 2、前端组件 × N。同样的代码，一遍遍手写。

前后端协调

后端改接口 → 前端改调用 → 联调 → 测试 → 上线。一个字段变更耗费半天。

权限系统

用户、角色、菜单、行列权限、按钮权限……每个项目都要重新实现一遍。

The Solution

注解即代码，代码即系统

Erupt 用注解描述业务意图，框架自动推导出完整的 CRUD、REST API、UI 渲染和权限管控。

SysUser.java
@Erupt(name = "用户管理") @Entity @Table(name = "sys_user") public class SysUser { @EruptField( views = @View(title = "用户名"), edit = @Edit( title = "用户名", notNull = true ) ) private String username; @EruptField( views = @View(title = "邮箱"), edit = @Edit(title = "邮箱", type = EditType.INPUT) ) private String email; } // ↑ 这就是全部代码

How It Works

运行机制

01

注解扫描

@EruptScan 启动时发现所有 @Erupt 类，解析为 EruptModel 元数据注册到上下文。

02

API 自动生成

框架根据元数据自动注册 REST 路由 /erupt-api/data/*，无需手写任何 Controller。

03

Schema 传输

后端将 UI Schema（字段类型、验证规则、布局）序列化发送前端，Angular 动态渲染表单与列表。

04

DataProxy 钩子

所有增删查改经过统一生命周期：beforeAdd、afterFetch 等，注入业务逻辑。

What You Get

核心能力

开箱即用的全套企业级能力，每一项都是生产可用的。

零代码 CRUD

增删查改、分页、批量操作、导入导出，注解声明即生效，无需一行实现代码。

RBAC 用户角色体系，支持菜单权限、行权限（数据隔离）、列权限、按钮级别控制。

rowPolicy = "org_id = #{orgId}" colPermission = @ColP(col="salary")

20+ 组件类型

INPUT、DATE、CHOICE、AUTO_COMPLETE、REFERENCE_TABLE、UPLOAD、CODE_EDITOR、MAP……字段类型决定渲染组件。

统一钩子注入业务逻辑，无需继承或代理类，只需实现接口方法。

Get Started

三分钟启动

从零到运行，只需初始化项目、配置数据库、启动服务。

01

初始化项目

访问 start.erupt.xyz 选择模块、数据库、版本，一键生成 Maven 项目骨架。

curl start.erupt.xyz/init \ -d "modules=upms,job,monitor"
02

配置数据源

在 application.yml 填写数据库连接，框架自动建表。

spring.datasource: url: jdbc:mysql://localhost/erupt username: root password: ****
03

启动 & 开发

运行 Spring Boot 主类，浏览器打开管理后台。新增 @Erupt 类即新增模块。

// 启动后访问 http://localhost:8080/
